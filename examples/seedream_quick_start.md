# Seedream 图生图节点快速测试

## 快速开始

### 1. 基础工作流

最简单的测试工作流包含以下节点：

```
[LoadImage] → [SeedreamImageToImageNode] → [SaveImage]
```

#### 节点配置示例：

**LoadImage 节点**:
- 加载任意一张图片

**Seedream 图生图节点**:
- `image`: 连接 LoadImage 的输出
- `prompt`: "转换为油画风格，色彩鲜艳"
- `api_key`: 你的火山引擎 API Key
- `model`: doubao-seedream-4.5
- `strength`: 0.6
- `size`: auto
- `seed`: -1
- `optimize_prompt_mode`: standard

**SaveImage 节点**:
- `images`: 连接 Seedream 节点的 image 输出

### 2. 对比工作流

对比原图和生成结果：

```
[LoadImage] ─┬→ [PreviewImage]
             └→ [SeedreamImageToImageNode] → [PreviewImage]
```

### 3. 批量测试不同强度

测试不同 strength 值的效果：

```
                          ┌→ [Seedream strength=0.3] → [SaveImage]
                          │
[LoadImage] ──────────────┼→ [Seedream strength=0.5] → [SaveImage]
                          │
                          └→ [Seedream strength=0.7] → [SaveImage]
```

### 4. 与其他节点组合

结合预处理和后处理：

```
[LoadImage] → [ImageResize] → [SeedreamImageToImageNode] → [ImageUpscale] → [SaveImage]
```

## 参数调优建议

### Strength 调优表

| Strength | 适用场景 | 效果 |
|----------|----------|------|
| 0.1-0.3 | 细微调整 | 保留原图大部分特征，只做轻微修改 |
| 0.3-0.5 | 风格微调 | 添加风格元素，保持主体内容 |
| 0.5-0.7 | 风格转换 | 明显的风格变化，部分保留原图 |
| 0.7-0.9 | 大幅重绘 | 根据提示词大幅修改 |
| 0.9-1.0 | 完全重新生成 | 几乎完全按提示词生成新图 |

### Size 选择建议

- **auto**: 保持原图尺寸（推荐）
- **1K**: 快速测试
- **2K**: 平衡质量和速度
- **4K**: 高质量输出（耗时较长）

## 预期处理时间

- **小图 (512x512)**: 约 5-15 秒
- **中图 (1024x1024)**: 约 10-30 秒
- **大图 (2048x2048)**: 约 20-60 秒
- **超大图 (4K+)**: 约 30-120 秒

*实际时间取决于网络状况和 API 服务器负载*

## 常见问题

### Q: 如何查看详细日志？
A: 在 ComfyUI 控制台窗口可以看到详细的运行日志，包括：
- 使用的参数
- API 调用状态
- 处理时间
- 错误信息

### Q: 生成的图片在哪里？
A: 如果使用了 SaveImage 节点，图片会保存在 ComfyUI 的 output 目录中。
默认路径: `ComfyUI/output/`

### Q: 如何固定生成结果？
A: 在第一次生成后，记录控制台输出的 seed 值，然后在节点中输入这个 seed 值即可重现相同结果。

### Q: 提示词不生效？
A: 尝试：
1. 使用更详细具体的描述
2. 启用 `optimize_prompt_mode` 为 `standard`
3. 增加 `strength` 值
4. 使用英文提示词（中文也支持，但英文可能效果更好）

## 技巧提示

💡 **批量处理**: 可以使用 ComfyUI 的批处理功能，一次处理多张图片

💡 **A/B 测试**: 使用相同的图片和不同的提示词，对比效果

💡 **渐进式调整**: 先用低 strength 测试，满意后再提高

💡 **保存原图**: 始终保留原始图片，方便对比和重新处理
